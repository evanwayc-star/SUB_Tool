# CHANGELOG — SUB Tool

所有版本的重要變更紀錄，依時間倒序排列。

---

## [v4.3.1] — 2026-07-01

### 修復
- **29.97 NDF 逐格跳格修復**：徹底解決在 29.97 專案下使用方向鍵或滾輪「逐格播放」時，因為影片時間小數點捨入誤差累積及播放器些微掉格所導致的「50:01 自動跳到 50:03」跳格現象。統一了所有精準幀率常數，並放寬了逐格同步的容錯度。

---

## [v4.3.0] — 2026-07-01

### 新增功能
- **字幕合併功能**：在右鍵選單中新增了「與上句合併」及「與下句合併」的快速功能。合併時會自動將兩句字幕文字串接（中間加上半型空白），並保留第一句的起點與第二句的終點。

### 修復
- **全選狀態同步**：修復了按下 `Ctrl + A` 全選字幕時，右下角狀態列與時間軸工具列未能正確顯示為「已選取 X 條」的 UI 更新遺漏問題。

---

## [v4.2.1] — 2026-07-01

### 修復
- **FPS 同步修正**：修復了 29.97 NDF 專案下，如果在特定影格按下 IN/OUT 點（如 `00:00:50:01`），因為內部取用了帶有浮點誤差的播放器時間而導致字幕時間碼無故多進位一格（變成 `00:00:50:02`）的問題。現在靜態取時已完全套用防抖修正，與畫面及時間軸刻度完美吻合。

---

## [v4.2.0] — 2026-06-30

### 新增與修復
- **調整視窗功能優化**：
  - 在「時間碼位移」與「調整持續時間」時，取消自動裁切重疊字幕的邏輯，單純移動 in 和 out 點。
  - 在調整持續時間時，若字幕原先已重疊，則保留原本設定不進行更動；若未重疊則自動貼齊下一句的 in 點。

## [v4.1.1] — 2026-06-30

### 新增與修復
- **介面用語調整**：全系統中的「備忘錄」文字已統一更名為「備註」，讓介面更簡潔一致。
- **快捷鍵顯示優化**：快捷鍵設定面板中的方向鍵現在會以直觀的箭頭符號（`↑`, `↓`, `←`, `→`）顯示，數字鍵盤則以 `Num` 開頭（如 `Num +`, `Num -`）顯示，提升可讀性。
- **預設快捷鍵配置更新**：
  - **放大時間軸**預設為：`2`、`=`（等同主鍵盤的 `+`）、數字鍵盤的 `+`。
  - **縮小時間軸**預設為：`1`、`-`、數字鍵盤的 `-`。
  - **切換時間軸縮放**預設為：反引號 (<code>\`</code>)、`\`、數字鍵盤的 `*`。

## [v4.1.0] — 2026-06-30

### 新增功能
- **上字幕模式狀態約束**：進入上字幕模式時，自動關閉「自動選取」、關閉「可覆蓋」、開啟「保留」，退出後自動恢復為先前的狀態。
- **上字幕模式列表凍結**：上字幕模式期間，字幕列表不再依 in 點重新排列，避免操作時 UI 跳動造成判斷錯亂；退出後自動重新排序。
- **上字幕模式序號固定**：上字幕模式下按 O 後自動前進的順序，依據進入模式前的字幕序號，而非即時時間排序。

### 修復
- **超長字幕自動清理（三層防護）**：修復上字幕模式中按 I 設定起點後，若未按 O 就切換或退出，字幕 end 會殘留在影片結尾（3605 秒）的問題。新增三層清理機制：
  1. 按 I 時立即清理其他 `_tempEnd` 字幕
  2. 退出模式時清理所有 `_tempEnd` 標記
  3. 安全網：退出時檢查所有曾被 I 鍵觸碰的字幕，若時長超過 10 分鐘則強制縮短
- **in 點在其他字幕範圍內的清理遺漏**：修復當 in 點落在其他字幕的 in/out 區間內時，`_tempEnd` 未被正確清理的問題。

## [v4.0.2] — 2026-06-30

### 新增與修復
- **時間碼位移計算修正**：修復了 `23.976` 等 NTSC NDF 影格率在「字幕調整視窗」與「時間軸快速修改」中輸入時間碼位移（如 `-01:00:00:00`）時，因秒數轉換誤差導致計算不精確的錯誤。現在已統一採用底層的核心精確換算，確保任何時間碼位移都能 100% 精準。

## [v4.0.0] — 2026-06-30

### 新增與修復

- **鎖定預設快捷鍵**：系統層級與基本文字編輯快捷鍵（如復原、重做、存檔、全選等）現已鎖定為不可修改，並以純文字呈現，避免誤觸修改。

## [v3.9.2] — 2026-06-30

### 新增與修復
- **次像素渲染與邊緣空隙修復**：修復了時間軸上位移 `left` 和寬度 `width` 在瀏覽器四捨五入計算時可能造成的 1px 渲染空隙，保證字幕區塊與標尺完全貼合。
- **拖曳對齊修正**：修正了在拖曳字幕右側邊緣吸附到播放點時，因系統錯誤錨定左側邊緣所引發的「1格 (frame) 誤差」現象。
- **I/O 按鍵時間精準度提升**：修復了使用 `In`、`Out` 鍵或 `+` 新增字幕時，由於尚未對齊格網造成的小誤差。現在所有的起迄時間都會強制經過影格對齊。
- **FPS 切換即時對齊**：修正了在變更 FPS 影格率設定時，時間軸與播放游標不會立即自動重繪的錯誤。切換 FPS 的瞬間，所有元素會立即磁吸至新的格網。

## [v3.9.0] — 2026-06-29

### 新增與修復
- **底層 SMPTE 影格精準度升級**：全面導入 NTSC 精確分數（如 `30000/1001`），淘汰所有小數浮點運算，確保 29.97 / 23.976 / 59.94 的時間轉換達到廣播級精確度。
- **時間尺刻度漂移修復**：修正時間軸在放大縮小時，刻度線因為浮點數誤差累積而偏離實際影格位置的問題。現在時間刻度與字幕邊緣會 100% 完美對齊。
- **強制格對齊機制**：無論是匯入 SRT 毫秒字幕或切換 FPS，系統都會自動掃描並將所有字幕精準吸附（Snap）至目前的影格格網，確保「最小單位為一格」，絕不卡在兩格之間。
- **專案載入穩定性修復**：修正載入舊專案時，若觸發非同步渲染可能導致影片與軌道遺失的內部錯誤。
- **快捷鍵擴充**：新增 `*` 鍵，與 `\` 鍵功能相同，可快速將時間軸縮放適配影片長度。

## [v2.8.0] — 2026-06-24

### 修復（程式碼審查 19 項）
- `_detectOverlaps`：改雙層掃描，修復長字幕包住多條短字幕時的漏報
- Web 自動備份：改存 `localStorage`（不再每 3 分鐘彈出下載對話框）
- ffmpeg.wasm CDN：優先 jsDelivr，fallback unpkg
- `Media.reset()`：清除 `_scrubEl` 子元素，修復跨檔案記憶體洩漏
- `extractAllAudio` ffmpeg FS：改 `try/finally`，確保失敗時清理虛擬 FS
- `_splitToChannelTracks` ID：改用遞增計數器，避免同毫秒碰撞
- `stopBufferSources`：`stop()` 後補 `disconnect()`，修復 AudioGraph 洩漏
- `History.record`：先做 O(1) 結構比對，再 fallback `JSON.stringify`，修復大量字幕時卡頓
- `drawTimeline` / `redrawTimeline`：拆分輕量版，減少播放路徑重繪開銷
- `mixer.js` 廢棄 analyser 參照：`reset()` 時呼叫 `clearMeterStrips()`
- `Wave.selectSource`：加 `myIdx` 快照防競爭條件
- `_bgAudioIngest` bail-out：清除 Audio 元素防背景預緩衝浪費
- `_durAdjCues`：改為接受 `scope` 參數，解除 DOM 緊耦合
- `sortCues`：加 `id` 次排序鍵，確保穩定排序
- `confirm()` 三處：改用 `openModal`，統一對話框風格
- `setTimeout(30ms)` 三處：改 `requestAnimationFrame`，符合瀏覽器渲染時序
- mpv bounds feeder：間隔從 1000ms 延長至 2000ms
- 大檔提示：500MB–1.6GB 與 > 1.6GB 給出不同說明
- `data.version || 1`：改明確檢查避免 `version: 0` 誤判

### FPS / 時碼一致性
- `encoreParts()`：提取為唯一「秒→時:分:秒:格」換算來源
- `fmtTick()`：改走 `encoreParts`，修復時間軸刻度與播放器時碼在 29.97 NDF 長時碼處差幾秒
- `nudge()`：改以 `displayTime()` 為基準，修復逐格跳兩格 / 退不動
- mpv `time-pos` handler：加 1.5 格抖動容忍死區，修復播放器比播放點多一格
- `pause` event：暫停時一併刷新 `tcCur/seekBar`，同源 `displayTime()`
- 新增 `FPS_時碼一致性.md`，記錄 6 大不變量與踩過的 3 個 Bug
