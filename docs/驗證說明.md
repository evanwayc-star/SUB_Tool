# 大型影音 / 混音器 — 桌面版驗證說明

驗證「mpv 秒開 / 單次讀取轉檔 + 逐聲道音訊 + 混音器電平表 + GPU 編碼 + 快取秒開」這批功能，請用桌面（Electron）版搭配真實大檔測試。

> **開檔有兩條路徑**：
> - 若系統偵測到 **mpv**：非原生格式或多音軌會以 mpv **秒開**（嵌入式覆蓋視窗播放），音軌於背景抽取；狀態列出現「啟動 mpv（秒開）…」。
> - 若無 mpv：走 **ffmpeg 單次讀取轉檔**（有進度 %），完成後以原生 `<video>` 播放。
> 兩種路徑的字幕編輯與混音器行為一致。

## 啟動方式

**驗證用（看得到錯誤，建議）** — 開兩個終端機：

```bash
# 終端機 1：vite dev server
npm run dev
# 終端機 2：Electron 連 dev server（會自動開 DevTools）
npm run electron:dev
```

**一般使用** — 直接雙擊 `啟動桌面版.bat`（build 後開 prod 版，無 DevTools）。

## 驗證清單

### 1. MXF（MPEG-2 4:2:2 + 8 單聲道）

- 開檔後狀態列依序：`讀取媒體資訊…` →「**讀取並轉檔中（單次讀取，大檔需數分鐘）…**」+ 進度 % → `媒體已載入（桌面模式）`
- [ ] 影片畫面出得來（修正前因 4:2:2 是黑的 / 不能播）
- [ ] 🔊 音軌面板出現正確聲道數量
- [ ] 整個來源檔只被讀取「一遍」（不再是 proxy + 每軌 + 波形分開讀很多遍）

### 2. MOV（ProRes HQ 4:2:2 10-bit + 5.1）

- [ ] 影片能播
- [ ] 🔊 出現 6 條聲道（5.1 被 channelsplit 拆成獨立聲道）

### 3. 混音器 🎚（時間軸工具列按鈕）

- [ ] 出現 N 條垂直電平表 + S（獨奏）/ M（靜音）
- [ ] 播放時：有內容的聲道表會跳動、無訊號的保持平的 → 逐聲道聽 + 看，決定要開哪些
- [ ] 表頭取「源頭訊號」：某聲道沒開（靜音）時，表仍會顯示它有沒有內容

### 4. 快取秒開

- [ ] 關掉該檔、再開同一支 → 狀態列出現「**使用既有快取，秒開…**」（或「**啟動 mpv（秒開）…**」）且幾乎瞬間載入
- 快取位置（讀取時依序找第一個有效的、寫入時找第一個可寫的）：
  ① 影片同資料夾的 `.subtool_Cache` ② `%APPDATA%\sub-tool\mediacache`（本機中央快取）
- 清除：時間軸工具列的 **🗂 快取管理** 對話框可「清理孤兒檔 / 全部清除」；或直接刪上述資料夾。
  （key = 路徑 + 大小 + 前 1MB 雜湊；來源檔內容變了會自動重轉）

### 5. 確認用到 GPU 編碼

- DevTools console 執行：`await window.subtool.status()`
- [ ] 回傳含 `venc: "h264_nvenc"`（或 qsv / amf；否則為 `libx264`）

### 6. FPS / 時碼一致性（播放器時碼 = 時間軸播放點）

> 背景與設計原則見 [`FPS_時碼一致性.md`](FPS_時碼一致性.md)；程式關鍵點以 `FPS-SYNC` 標記。建議用 **29.97fps NDF** 的長片（一小時以上）測，最容易暴露問題。

- [ ] **同格**：seek 到任一字幕起點並暫停 → 播放器大時碼 與 時間軸播放點正下方刻度的時碼**完全相同**（不差一格）。
- [ ] **逐格精度**：在高時碼處（如 01:20:00 之後）按 `→` 一次只前進**一格**、按 `←` 一次只後退**一格**（不可跳兩格、不可按左鍵又彈回原位）。
- [ ] **刻度一致**：時間軸放大到顯示「格」層級時，播放點所在刻度標籤 與 播放器時碼一致；字幕列表的 In/Out 時碼也與之同格。
- [ ] **mpv 沉降**：以 mpv 路徑開檔，seek 後暫停，反覆數次 → 播放器時碼不會偶爾比播放點多/少一格。
- [ ] **（正常現象，非 bug）** 29.97 **NDF** 在長時碼處，播放器時碼會比真實經過秒數**慢幾秒**（每小時約 3.6 秒）——這是非 drop-frame 時碼的標準特性；若需時碼≈牆鐘，請改用 **29.97 (DF)**。

### 7. 空白專案基本流程

- [ ] 新專案啟動後，時間軸已有「軌道 1」（不需手動新增）
- [ ] 無影片時可拖曳播放點（播放點不卡在 0）
- [ ] 按 `I`/`O` 可在目前位置新增空白字幕並出現在時間軸
- [ ] 第一次匯入影片或字幕時，時間軸自動縮放至完整影片長度

### 8. 專案載入（桌面版特有）

- [ ] 雙擊 `.subtool` 檔案：軟體可自動啟動並載入該專案（包含影片與字幕進度）。

## 出問題怎麼回報

- **DevTools console** = renderer（前端）錯誤
- **終端機 2（Electron）** = main 行程 / ffmpeg 錯誤
- 把紅字訊息貼出來即可

## 預期效能

- **第一次**開檔會等單次轉碼完成（GPU 下 MXF 約 4–5 分、MOV 約 7–12 分，主要受讀取大檔的硬碟 I/O 限制）。
- **第二次以後**同一支片由快取秒開。

## 設計上的取捨（已知、非 bug）

- 採「單次讀取 + 進度條 + 快取」，**沒有**做「轉檔途中即可編輯」（那需要 HLS 串流，會造成時間軸累積漂移 5.9–7.4 秒，對逐格字幕是大忌，故不採用）。
- 時間軸波形為單一混音波形（目前不隨獨奏切換，逐聲道判斷請用混音器電平表）。
- 聲道分組（5.1 / 2.0）無法從中繼資料可靠判斷（檔案 channel_layout 常為 unknown），故一律逐聲道攤開，由使用者自行判斷。
