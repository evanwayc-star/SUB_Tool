# SUB Tool — 開發 / 建置說明（架構：Vite + ES 模組 + 事件匯流排）

> **📌 程式碼品質與效能審查報告：** 詳見 [project_review.md](../project_review.md)

本專案為 **Vite + 原生 ES 模組**（無框架），打包後輸出**單一可雙擊的 HTML**（`vite-plugin-singlefile` 把 JS/CSS 全部內嵌），兼顧可維護性與「免安裝、點開即用」。同一份程式碼同時跑網頁版與 Electron 桌面版（以 `window.subtool` / `IS_DESKTOP` 分支）。

## 指令

```bash
npm install           # 第一次（安裝 Vite / Electron 等）
npm run dev           # 開發：本機伺服器 + 熱更新 (HMR)，開 http://localhost:8777
npm run build         # 打包成單一檔 dist/index.html（內嵌 JS/CSS，可直接雙擊）
npm run preview       # 預覽打包結果
npm run electron      # 打包後以 Electron 桌面版開啟
npm run electron:dev  # Electron 連 dev server（會開 DevTools，驗證用）
npm run dist          # 打包 + electron-builder 產生安裝檔 (exe)（win target）
```

一般使用者：雙擊 `啟動工具.bat`（自動 build 並開啟 dist 單一檔）或 `啟動桌面版.bat`（build + Electron）。

## 架構：事件匯流排（解除與 app.js 的雙向相依）

**重點：沒有任何模組 `import … from './app.js'`。** 低階模組需要觸發「重繪 / 指令」時，呼叫 `emit('事件名', …)`（來自 `events.js`），**不直接 import 渲染函式**。`app.js` 於啟動時以 `on('事件名', 處理函式)` 訂閱，收到事件後呼叫對應的渲染/指令函式。`emit` 為**同步**呼叫，語意與直接呼叫原函式完全一致。

目前的事件：

```
render:all         → renderAll            render:videoSub    → renderVideoSub
render:listTrackSel → renderListTrackSel   render:trackStyle  → renderTrackStyle
playhead:ensure    → ensurePlayheadVisible  duration:known    → onDurationKnown
mpv:refreshSubs    → refreshMpvSubs        panel:toggle      → togglePanel(id)
note:openInPanel   → openNoteInPanel(n)    cue:openEdit      → openCueEditModal(c)
action             → doAction(act)
```

其餘跨模組相依改為「就近直接 import」而非繞經 app.js：基本 UI 一律 `import … from './ui.js'`；磁吸工具定義在 `timeline.js`；`detectFpsWeb` 定義在 `media.js`。

**維護守則：不要再新增 `import … from './app.js'`。** 需要 app.js 協調的動作，請改用 `emit(...)`，並在 app.js 的訂閱區補一行 `on(...)`。

## 驗證 / 拆分安全網

- `npx eslint src`：抓「使用了未匯入的名稱」(no-undef)。
- `npm run build`：rollup 會抓「import 了某模組未 export 的名稱」（eslint 抓不到 export 端）。
- 確認沒有重新引入循環：`grep -rn "from './app.js'" src` 應為**空**。
- 開發期 `import.meta.env.DEV` 為真時會掛上 `window.SUB` 除錯把手（State/Media/各函式），可從瀏覽器 console 或 preview_eval 取用；正式 build 自動移除。
